# Directive: n8n Self-Annealing Error Dashboard

## Goal
Create a system that monitors n8n for failed workflow executions, attempts to automatically fix them using available tools, and displays the status on a web dashboard.

## Components
1.  **Monitor & Healer Engine** (`execution/monitor_and_heal.py`):
    -   Polls n8n for executions with `status='error'`.
    -   Extracts error details.
    -   Attempts to fix using `n8n_autofix_workflow` or standard debugging heuristics.
    -   Logs events to `.dashboard/data/events.json`.

2.  **Web Dashboard** (`dashboard/`):
    -   React + Vite application.
    -   Displays a list of recent errors.
    -   Shows "Auto-Fix Attempted" status and resolution time.
    -   Provides specific error explanations.

## Inputs
-   n8n Instance Connection (Existing MCP Tool).
-   `n8n-mcp` tools: `n8n_executions`, `n8n_autofix_workflow`.

## Output
-   A running web dashboard at `http://localhost:5173`.
-   A JSON log of all self-healing attempts.

## Process
1.  **Initialize**: Setup React project in `dashboard/`.
2.  **Monitor**: Script runs every X minutes (or triggered manually).
3.  **Analyze**:
    -   Get failed execution ID.
    -   Get Workflow ID.
    -   Run `n8n_autofix_workflow` (preview mode first).
    -   If high confidence, apply fix.
4.  **Report**: Update `events.json`.
5.  **Visualize**: Dashboard auto-refreshes data.

## Edge Cases
-   **Unfixable Errors**: Log as "Requires Manual Intervention" with a suggested solution generated by AI.
-   **Connection Failure**: Alert specific visible status on dashboard.
